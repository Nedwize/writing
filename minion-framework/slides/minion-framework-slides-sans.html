The Insight

Make a SQLite table the orchestrator.

No orchestrating Claude. No script. The agents don't talk to each other — they just race for rows.

Agents · Claude Code · Fine-tuning2026-06-05
04 · What workedThe Minion Framework
One clause makes it safe

Atomic claim, no locks, no Redis, no Celery.

UPDATE work_queue
SET    status = 'claimed', claimed_by = :wid
WHERE  thread_id = :tid
AND    status = 'queued';  -- only one Army wins

Sixteen Armies run the same UPDATE. Only one matches. Just a table.

05 · The solutionThe Minion Framework
Three primitives

That's the whole framework.

1
A shared SQLite work queue
Every thread is a row. WAL mode, 30-second busy timeout. queued → claimed → done.
2
N parallel Minion Armies
Each Army is a terminal running claude on a /goal loop: claim ten IDs, work, repeat.
3
Per-Army minion fan-out
Ten ephemeral sub-agents per dispatch — one thread each, write JSON, exit. Peak: ~160 in flight.
Agents · Claude Code · Fine-tuning2026-06-05
05 · The solutionThe Minion Framework
The seventeenth session

Gru watches the fleet.

Sixteen Armies is too many to track by hand. Gru runs a /loop heartbeat every five minutes — queue depth, per-Army throughput, context windows, and the one thing that mattered most: the 5-hour plan budget.

Gru doesn't orchestrate. Gru observes — and tells me when to slow down before the 429s hit.

Each wins somewhere.

Minion still wins
  • No concurrency cap — the limit is your plan, not the runtime
  • Resume survives anything: reboots, crashes, two laptops over three weeks
  • Cross-machine fan-out is a network mount away
Workflows just better

  • Observability you don't have to build or debug
  • The runtime paces your plan — no budget babysitting

The takeawayThe Minion Framework
The takeaway

What you build the orchestrator out of is a design choice, not a default.

A JS runtime, or a SQLite table and a boss named Gru. Both beat the one thing you should never do: put the plan in a Claude context window and watch it forget what it was doing at 100K tokens in.
